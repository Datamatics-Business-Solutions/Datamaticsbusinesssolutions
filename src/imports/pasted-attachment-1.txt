Perform a functional improvement audit 
on the entire platform. 

IMPORTANT RULES:
- Do NOT change any colors
- Do NOT change any fonts or font sizes
- Do NOT change any existing layouts
- Do NOT change any card designs
- Do NOT change any existing styles
- ONLY add the functional improvements 
  listed below

---

## 1 — EMPTY STATES

Every table and list must show a helpful 
message when there is no data.
Right now they show nothing or look broken.

Add an empty state to:
- Campaigns table
- Leads table
- Invoices table
- Documents table
- Support tickets table
- Recent Activity feed

Each empty state:
- Centered in the empty area
- Small gray Lucide icon (32px)
- Short title: "No [items] yet"
- One line subtitle explaining where 
  data will appear
- Action button only where it makes sense

---

## 2 — LOADING STATES

Every page must show a skeleton 
while data is loading.
Never show a blank white screen.

Add shimmer skeleton placeholders for:
- KPI cards (gray shimmer rectangles)
- Tables (5 gray shimmer rows)
- Charts (gray rectangle placeholder)
- Activity feed (4 shimmer rows)

---

## 3 — BUTTON FEEDBACK

Every button must give immediate 
visual feedback on click:

- On click: scale(0.97) for 100ms
- Save buttons: show a loading spinner 
  while processing, then a brief 
  checkmark on success
- Destructive buttons: brief shake 
  animation on error

---

## 4 — TOOLTIPS ON ICONS

Every icon-only button in tables 
(View, Download, Edit) must show 
a tooltip on hover with the action name.

Tooltip appears after 300ms hover delay.
Use the existing tooltip style already 
in the app.

---

## 5 — SEARCH AND FILTER IMPROVEMENTS

All search inputs must:
- Wait 300ms after typing stops 
  before filtering (debounce)
- Show a small clear (X) button 
  when text is entered
- Show "No results found for [term]" 
  when search returns nothing

---

## 6 — TABLE IMPROVEMENTS

Do not change any visual styling.
Only fix these functional issues:

- Clicking a column header should 
  sort that column (ascending/descending)
- Show a small sort arrow icon next 
  to the active sort column
- Rows should be clickable and 
  navigate to the detail view 
  where relevant (leads, invoices, 
  campaigns, support tickets)
- Checkboxes in tables should have 
  a "select all" checkbox in the header

---

## 7 — FORM VALIDATION (Account Page)

Add basic validation to all form fields:

- Required fields show a red border 
  and inline error message if left empty
- Email field validates email format
- Phone field validates phone format
- Show "Changes saved successfully" 
  confirmation after saving
- Show "Failed to save — please try again" 
  on error

---

## 8 — NAVIGATION IMPROVEMENTS

- Browser back button must work 
  correctly on all pages
- Active page in sidebar must always 
  match the current URL
- Page title in the browser tab must 
  match the current page name:
  "Dashboard — Datamatics"
  "Campaigns — Datamatics"
  "Leads — Datamatics"
  etc.

---

## 9 — MOBILE NAVIGATION

On screens under 768px:
- Sidebar is hidden by default
- Show a hamburger icon (☰) 
  in the top left of every page header
- Tapping it slides the sidebar in 
  as an overlay
- Tapping outside closes it

- Fix every NaN value remaining

- Fix every "Invalid Date" remaining

- Ensure no console errors appear
